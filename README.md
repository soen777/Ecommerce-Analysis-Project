# 淘宝用户行为数据分析与关联规则挖掘

> 广东财经大学 · 数据科学与大数据技术（大一）· 暑期实习项目

本项目基于阿里天池公开的淘宝用户行为数据集（约1亿条记录），完成**数据清洗 → 漏斗分析 → 用户分层（RFM+KMeans）→ 商品关联规则挖掘（FP-Growth）→ 销量预测 → 可视化看板**的全流程分析，模拟真实电商数据分析任务。

---

## 🎯 项目背景

- **数据来源**：阿里天池“淘宝用户行为数据集”（2017-11-25 ~ 2017-12-03）
- **原始数据量**：约 1 亿条用户行为（点击、收藏、加购、支付）
- **分析目标**：
  - 掌握用户行为转化漏斗
  - 识别高价值用户群体
  - 发现商品之间的强关联规则，用于交叉销售
  - 预测未来日销量

## 📊 核心分析步骤与结果

### 1. 数据清洗
- 处理缺失值、重复值，转换时间戳 → 日期/小时
- 过滤超出日期范围（2017-11-25 ~ 2017-12-03）的数据
- **最终有效数据**：约 9890 万条

### 2. 用户行为漏斗分析
| 行为 | 数量 | 转化率（相对于点击） |
|------|------|----------------------|
| 点击（pv） | 89,000,000+ | 100% |
| 加购 + 收藏 | 约 5,600,000 | ~6.3% |
| 支付（buy） | 约 1,200,000 | **1.35%** |

> 结论：加购到支付环节流失严重，可针对性优化购物车提示或优惠券策略。

### 3. RFM + KMeans 用户分层
- **R**：最后一次购买距今天数（基准日 2017-12-04）
- **F**：购买次数（无金额数据，用频次代替 M）
- **聚类结果**（3类）：

| 用户类型 | 占比 | 运营建议 |
|----------|------|----------|
| 高价值用户 | 8% | VIP维护、专属优惠 |
| 潜力用户 | 22% | 推送加购未支付提醒 |
| 流失用户 | 70% | 大额召回券 |

### 4. FP-Growth 商品关联规则
- 过滤低频商品（购买人数 ≥ 200）
- 抽样 4.3 万用户，最小支持度 0.00116（至少50人）
- 挖掘到 **166 个频繁项集**，生成关联规则（置信度≥0.2）

**示例规则**：
| 前件 | 后件 | 置信度 | 提升度 |
|------|------|--------|--------|
| 商品1042152 | 商品5062984 | 29.8% | **38.0** |
| 商品1042152 | 商品5051027 | 23.7% | 35.1 |

> 提升度 > 1 表示正相关，提升度 38 说明购买前件后购买后件的概率是随机情况的 38 倍，极具推荐价值。

### 5. 日销量预测（线性回归 + 周末效应）
- 以日期序号和是否周末为特征，预测次日销量
- **预测次日销量**：约 342 单（实际值需对比，此处为演示）
